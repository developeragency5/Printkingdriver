Print Technology

CUPS Explained

A clear explanation of cups explained for non-engineers.

Quick Summary

CUPS (Common Unix Printing System) is the printing system used by macOS, Linux, and other Unix-based operating systems. It manages all communication between applications and printers, converts documents to the format each printer understands, and handles the print queue. CUPS was acquired by Apple in 2007 and is now the backbone of printing on both macOS and Linux.

  • CUPS provides a web interface at http://localhost:631 for managing printers and queues
  • macOS uses CUPS internally for all printing — AirPrint and IPP Everywhere are layered on top of it
  • On Linux, CUPS is installed by default in most distributions and managed via the same web interface or the desktop printer tool

Overview

CUPS is the print spooler used on macOS and most Linux distributions. It manages print queues, handles job spooling and provides a web-based admin interface.

What CUPS Does

CUPS receives print jobs from applications, queues them, processes them through the right driver (called a PPD on CUPS), and sends them to the printer. It is the macOS and Linux equivalent of the Windows Print Spooler.

The Web Interface

Browse to localhost:631 on any macOS or Linux machine to access the CUPS admin interface. From there you can add, delete and configure printers without going through the OS settings dialog.

Config Files

CUPS configuration lives in /etc/cups/. The main file is cupsd.conf. Each printer has its own .ppd file in /etc/cups/ppd/ that describes the printer's capabilities.

Common Commands

lpstat -p — list all configured printers. lpq — show the current queue. cancel JOB-ID — cancel a job. lpadmin — add or modify printers. systemctl restart cups — restart the service on Linux.

Troubleshooting

If printing breaks, check /var/log/cups/error_log. Most issues are visible there. If the queue freezes, restart CUPS — the equivalent of restarting the Windows Print Spooler.

Bottom Line

Anyone running macOS or Linux is using CUPS whether they realise it or not. Knowing where the config lives saves hours when something breaks.
