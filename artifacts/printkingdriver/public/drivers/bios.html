Advanced System

BIOS / UEFI Driver

Initialises hardware before the OS loads.

What Is It?

Firmware initialises hardware before the operating system loads. It ensures all components are ready and configured properly.

It controls boot order, hardware settings, and security features. Updates improve compatibility and system stability.

How It Works

When power is applied, the firmware checks each major component, configures memory and CPU features, and selects a boot device. It then loads the OS bootloader and provides a minimal interface the OS uses until its own drivers are ready.

Operating System
Firmware Driver

Key Functions

  • Performs the power-on self-test (POST).
  • Initialises memory, CPU features, and storage controllers.
  • Selects and launches the boot loader.
  • Provides runtime services for time, NVRAM, and secure boot.

Why It Matters

Without functioning firmware, the computer never reaches the operating system. Firmware quality affects boot speed, hardware compatibility, and platform security.

Best Practices

A short checklist to keep this driver healthy and reduce the chance of running into the issues above.

  • Plug the system into mains power before flashing — power loss mid-update can brick the board.
  • Read the change-log; flash only when a fix or feature is genuinely needed.
  • Note your current firmware settings (memory profile, fan curves, boot order) before updating.
  • Use the manufacturer’s official tool, not third-party flashers.
  • After flashing, re-enable virtualisation, the security module and secure boot if you rely on them.
