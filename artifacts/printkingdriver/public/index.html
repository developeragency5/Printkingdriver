Explore Drivers

Find and learn about all essential system drivers, organised into clear categories for easy reference.

01 Category 01

Essential Drivers

Core system drivers required for basic operation. They allow the OS to recognise and use the most fundamental components of a computer.

Includes4 drivers
  • Chipset Driver
    Coordinates motherboard logic and CPU communication.
  • Graphics Driver
    Handles visual rendering and GPU instructions.
  • Audio Driver
    Manages sound input and output processing.
  • Network Driver
    Enables wired and wireless network connectivity.
02 Category 02

Hardware-Specific Drivers

Drivers that control specialised internal hardware. They enable the OS to fully use each component's features and performance.

Includes4 drivers
  • Storage Controller
    Manages data transfer with SSDs and hard drives.
  • USB Driver
    Detects ports and connected USB devices.
  • Bluetooth Driver
    Pairs and manages nearby wireless devices.
  • Touchpad / Keyboard
    Translates input gestures and keystrokes.
03 Category 03

Peripheral Drivers

Drivers for external devices that connect to a computer. They define how peripherals exchange data with the host system.

Includes4 drivers
  • Printer Driver
    Translates documents into printer commands.
  • Scanner Driver
    Captures images and handles OCR exchange.
  • Webcam Driver
    Processes video stream capture and resolution.
  • Display Driver
    Controls external monitor signal output.
04 Category 04

Advanced Systems

Low-level firmware and security drivers that operate close to the hardware and influence overall platform integrity.

Includes4 drivers
  • BIOS / UEFI
    Initialises hardware before the OS loads.
  • Security Driver
    Supports encryption and secure boot.
  • Monitor Calibration
    Manages colour accuracy and profiles.
  • Power Management
    Coordinates CPU and battery efficiency.

Concept

How Drivers Work

A driver is the bridge between physical hardware and the software running on the operating system. The diagram below shows the typical flow of communication.

Hardware
Physical device or component
Driver
Software translator layer
Operating System
Manages resources & I/O
Application
User-facing program
When an application makes a request, the operating system routes it to the correct Driver, which translates it into instructions the hardware can execute. The result then travels back up the chain to the user.
Reference

Types of Drivers and Their Roles

A summary of common driver types, the function each one performs, and a typical use case in everyday computing.

Driver TypeFunctionExample Use Case
Graphics DriverTranslates rendering commands into GPU instructions.Display visuals, video playback, gaming.
Audio DriverManages sound signal processing between OS and audio chip.Music playback, voice calls, system sound.
Network DriverHandles data packet exchange over wired or wireless links.Internet access, file sharing, streaming.
Storage ControllerCoordinates read and write operations to internal storage.Booting the OS, saving files, data caching.
USB DriverDetects and routes data for connected USB devices.Flash drives, keyboards, mice, peripherals.
Printer DriverConverts digital documents into printer-readable commands.Office printing, design output, scanning.
BIOS / UEFIInitialises hardware components before the OS loads.System boot sequence and firmware updates.
