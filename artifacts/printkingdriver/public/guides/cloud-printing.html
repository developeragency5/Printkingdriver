Cloud Printing Overview

Everything that has to line up for a cloud printing overview to work first time, and what to do when one of those things does not.

Quick Summary

Cloud printing sends a print job from any device, anywhere in the world, to a cloud-connected device — without both being on the same network. Google Cloud Print was retired in 2021. Modern equivalents include the manufacturer's Smart, the manufacturer's Connect, the manufacturer iPrint&Scan, and printing through email addresses assigned to the device by the manufacturer.

  • Google Cloud Print was shut down in December 2020 — it no longer works
  • major manufacturersall have their own cloud printing services requiring account registration
  • Most cloud services assign a unique email address to the device — send a file to that address to print it remotely

Overview

Cloud printing lets you send a job to a device from any internet-connected device, even when you are not on the same network as the device. Each manufacturer offers their own cloud-print service, and there are also vendor-neutral options like Mopria Cloud and the older Google Cloud Print (now retired).

Step-by-Step

  • Confirm the device is cloud-capable — Look in the device's network menu for a Cloud or Web Services section. If it lists a service like the manufacturer's ePrint, the manufacturer's Connect or the manufacturer's Cloud, the device can join.
  • Register the device with the vendor cloud — From the device's web interface or vendor app, sign in to your vendor account and register the device. The vendor assigns a unique email address or service ID to the device.
  • Enable on the sending device — Add the cloud device to your phone or laptop using the vendor app. The job is uploaded to the vendor cloud and forwarded to the device.
  • Watch the security model — Some cloud services accept jobs from any sender that knows the device email address. Restrict accepted senders in the cloud account if the device is on a public network.
  • Have a fallback — Cloud services occasionally go down. For business-critical devices, also configure direct network access so users can print without the cloud round trip.

What Can Go Wrong

A short list of the issues people most often run into during cloud printing overview, and the right fix for each:

SymptomLikely CauseFirst Step
Connection succeeds but no printDriver mismatched to the model.Re-add the vendor driver.
Connection drops after a few minutesPower-saving or sleep settings.Disable sleep on the device or computer.
Device not discoveredDiscovery protocol blocked or different subnet.Confirm both devices on same network.
