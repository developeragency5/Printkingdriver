Setup Guide

Linux Device Setup

Everything you need to add a device on Linux, choose the right driver and avoid the set up errors people most often hit.

Quick Summary

Linux device setup uses CUPS — the Common Unix Printing System — to discover and manage devices. Most modern distributions (Ubuntu, Fedora, Debian, Mint) add USB and network devices automatically through the GNOME or KDE device settings tool. Headless servers and specialist devices may require terminal commands and manual PPD file setup.

  • GNOME: Settings > Devices; KDE: System Settings > Devices — both use CUPS underneath
  • For headless setups, use the CUPS web interface at http://localhost:631
  • Some brands (major manufacturers) provide Linux driver packages (.deb or .rpm) on their support sites

Before You Start

Setting up a device on Linux is usually painless if you start from a clean state. Make sure Linux itself is fully updated, the device is powered on and within reach of the network you intend to use it on, and you know whether you want to connect by USB, Ethernet or Wi-Fi. Mixing connection types mid-set up is the most common cause of a stuck setup.

Most distributions auto-detect IPP-Everywhere devices without any vendor driver. Set Up the vendor PPD only when you need finishing options the generic driver omits.

Step 1 — Open Device Settings

Open Settings » Devices (GNOME) or System Settings » Devices (KDE). This is where every device-related task on Linux starts. Click Add a device or scanner (or the equivalent on your distribution).

Linux will scan the local network for any device that supports the modern CUPS with PPD or driverless IPP stack. Most devices from the last six years will appear within 30 seconds. If yours does not, move on to step two.

Step 2 — Choose How to Add the Device

Linux offers several ways to add a device. The right choice depends on the connection:

  • Wireless / network device — pick the device from the auto-discovered list. Linux downloads the right driver automatically using CUPS with PPD or driverless IPP.
  • USB device — connect the cable. Linux should detect it and set up the driver in 10–30 seconds.
  • Manual add by IP — choose "Add by IP address" or the equivalent and enter the device's IP. Use this when the device is on a different subnet to your computer.
  • Vendor setup tool — download the manufacturer's package when you need scanning, faxing or any vendor-specific feature beyond basic print.

Step 3 — Pick the Driver

After Linux has identified the device, it will offer a driver. You usually have three options:

Driver ChoiceWhat You GetWhen to Use
Built-in / genericPrint only — no scan, no fax, no ink-level monitor.Quick setup, occasional printing.
Vendor full packagePrint, scan, fax, status monitor, vendor app integration.Multi-function devices used daily.
PostScript or PCLPage-description language driver for graphics or office workloads.Design studios, large workgroups.

For most users at home or in a small office, the vendor full package is the right choice. See our brand pages for where to download the package for your specific device.

Step 4 — Print a Test Page

Once the driver is set up, Linux adds the device to the queue list with a status of "Ready". Right-click (or long-press on touch devices) and choose Print test page. The test page confirms three things: the connection is alive, the driver is rendering correctly, and the device itself is in working order.

If the test page does not arrive, it is almost always a connection issue rather than a driver problem. Check the device offline guide for the next steps.

SymptomCauseFix

"Driver unavailable" messageLinux could not download the driver from its update service.Run Linux updates first, retry. If still failing, set up the vendor driver.
Device adds but won't printWrong driver matched to the model.Remove the device, set up the vendor driver, add it again.

After Setup — Recommended Settings

Once the device prints, three quick changes will save you trouble later:

  • Set the right paper size as default. Mismatched defaults (US Letter vs A4) cause silent crops on every job.
  • Set print quality to Normal, not Best or Photo. The visual difference is small for everyday work; the speed and ink saving is large.
  • Disable any auto-update tool the vendor sets up unless you actively want it. Quietly self-updating drivers cause more problems than they solve.
  • For network devices, set a static IP at the router so the print queue does not break when the IP changes.
